TL;DR: Login, configure your key at , start Mining, click , click on a Player result.

Button Functions
  • Performs a search on the player database based on specified query parameters
  • Configure query parameters (every field but the first can be left blank)
  • Mining: Start/Stop scanning players' data
  • Configure your Public API key
  • Open this window
  • Login: ...

The indicator shows how much credits you have. Mining one player's data earns one credit and searching costs two credits per search result. If possible, leave Mining running in the background to help the site gather new PVP targets.

Clicking on a search result will be registered as a Hit, opening the player's Attack page. A Hit will impact the 3x/4x filtering of your search results.

PVPTool Guide (PC only)
  • Install Tampermonkey (NOT Greasemonkey or Violentmonkey) for your browser
  • Install the PVP Tool userscript
  • Close ALL SMMO tabs
  • Close ALL PVP Tool tabs (except for this one) and refresh this page
  • Click on a player hit
  • Now you press spacebar on the Attack page to switch to the next hit
PointsListener Guide (PC only)
  • Install Tampermonkey (NOT Greasemonkey or Violentmonkey) for your browser
  • Install the PointsListener userscript
  • Replace DISCORD_UID (and the webhook URL if you like)
  • Close all SMMO tabs then open a new SMMO tab
  • Now whenever you click the avatar button on the top right, the scheduled timing for the notification will be refreshed
  • Keep any SMMO tab open to receive Discord notifications
  • SMMO Website integration via Userscript (PC only)
  • Press spacebar to auto-hit/auto-query
  • 3x/4x player filtering based on hit logs
  • Automated mining: the tab can be left in the background and mining will proceed
  • Distributed mining: public API requests are performed client-side but stored on server, enabling for aggregate search results
  • Optimized search: probability of a player's data being refreshed is proportional to number of hits logged
  • Player refreshing: after a player is hit, a refresh of the player's data is scheduled to happen after revival (>50% health)
  • Mobile device compability
  • Dark mode: self-explanatory

This website is built with a Bootstrap frontend, Flask REST API, and a PostgreSQL database.

If you are interested in building your own client atop the API, feel free to use these clients (JS or Python) as reference points.
